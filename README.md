## Aiotto 是什么

Codex 本身专注于完成任务，但长期使用后，日常维护会逐渐散落在账号、额度、模型服务、会话记录、本地存储和扩展配置之间。用户往往知道“哪里出了问题”，却不知道应该先检查什么、修改会影响什么、失败后又该怎样恢复。

Aiotto 是为 Codex 桌面用户设计的本地工作台。它把高频状态集中展示，把重复操作整理成清晰流程，并在账号切换、配置修改、会话清理和数据恢复等高风险动作前提供确认、备份与结果复检。

Aiotto 特别适合：

- 同时使用多个 Codex 账号，需要关注 5 小时和每周额度的用户。
- 希望在官方模型与第三方模型服务之间灵活选择的用户。
- 积累了大量历史会话，需要搜索、整理、恢复和控制磁盘占用的用户。
- 使用 Skills、MCP，并希望快速了解本机扩展状态的用户。
- 希望不打开主窗口，也能从 macOS 状态栏掌握关键信息的用户。

## Aiotto 解决了哪些痛点

### 1. 多账号并不难，难的是“不知道哪个账号现在能用”

**常见问题**

- 5 小时额度和每周额度分散，任务中断后才发现账号已经触顶。
- 多个账号的套餐、凭据健康度、重置时间和当前登录状态难以放在一起比较。
- 手动切换账号容易忘记备份，也不容易确认切换是否真正生效。

**Aiotto 的处理方式**

- 在同一页面展示账号状态、套餐、额度窗口、重置时间和健康提示。
- 支持保存、导入、导出和整理账号快照，并在切换前明确提示影响。
- 提供额度阈值提醒、按需开启的账号自动切换和五小时额度计划，降低任务被额度突然打断的概率。
- 切换完成后刷新工作台状态，并通过明确反馈告诉你成功、失败或需要重新登录。

### 2. 第三方模型能用，不代表配置过程可靠

**常见问题**

- Base URL、API Key、协议和模型名称只要有一项不匹配，就可能得到难以理解的错误。
- 不知道服务是否可连接、模型是否存在、余额是否充足，往往要真正发起任务后才发现问题。
- 切换模型服务时，用户还希望保留官方账号能力、历史会话和随时恢复原状态的路径。

**Aiotto 的处理方式**

- 集中管理 OpenAI-compatible 与 Anthropic-compatible 模型服务、模型列表、余额、延迟和启用状态。
- 支持模型拉取、连通性测试、首包耗时、网络模式、错误解释和请求日志导出。
- 支持中转配置的添加、编辑、删除、导入和导出，导入前先预览结果。
- 通过 Codex 智能路由在官方模型与中转模型之间选择；关闭后可以回到正常的官方使用路径。
- 对不支持生图工具的模型服务提供兼容选项，减少因工具能力不匹配导致的请求失败。

### 3. 会话越来越多后，“找到、继续和安全清理”都会变难

**常见问题**

- 历史会话数量增长后，仅靠原生列表很难快速找到目标任务。
- 长会话容易从错误位置开始阅读，用户消息、AI 回复和内部记录也不容易区分。
- 会话文件、归档数据和临时内容持续占用空间，但直接删除又担心丢失重要记录。

**Aiotto 的处理方式**

- 支持浏览、搜索、筛选、置顶和继续本地 Codex 会话。
- 提供更清晰的消息角色、长会话目录和响应式阅读布局。
- 支持归档、回收、恢复和导出；删除类操作提供确认与恢复路径。
- 展示本地存储构成、会话文件占用和可清理建议，先预览再处理，完成后重新扫描验证结果。

### 4. 用量不透明，就很难判断钱和时间花在了哪里

**常见问题**

- 只看到任务完成，却不知道输入、输出、缓存和成本分别是多少。
- 官方与中转请求混在一起，不容易判断哪个模型或服务消耗更多。
- 模型价格变化后，历史统计与当前成本容易失去参考价值。

**Aiotto 的处理方式**

- 汇总请求量、输入输出 token、缓存读写、模型来源和成本趋势。
- 区分官方与中转使用情况，并按时间范围、模型和来源查看明细。
- 提供模型价格信息和成本计算结果，让额度使用和费用变化更容易解释。

### 5. 真正让人不安的不是配置多，而是“改完以后无法恢复”

**常见问题**

- Skills、MCP、账号设置和模型服务配置分散，出现异常时难以判断是哪一处引起的。
- 清理、修复、账号切换和恢复都可能改变本地状态，直接尝试风险很高。
- Codex 或 OpenAI 服务异常时，用户经常分不清是本机配置问题、账号问题还是服务事故。

**Aiotto 的处理方式**

- 盘点 Skills 与 MCP 的来源、状态、基础信息和健康情况。
- 通过备份中心查看历史记录、恢复入口和导出位置。
- 系统诊断默认先做只读检查，再针对明确异常提供处理入口；需要修改时先确认影响，并在完成后复检。
- 独立展示 OpenAI、ChatGPT 与 Codex 服务状态和事故进展，帮助区分本机问题与外部服务异常。

## 核心能力

### 账号与额度

- 多账号状态、套餐、凭据健康度和当前登录账号集中展示。
- 5 小时额度、每周额度、重置时间和限额重置额度统一查看。
- 保存当前登录账号，导入、导出、删除和切换账号快照。
- 额度刷新频率、阈值提醒、自动切换和五小时额度计划。
- 安全登出与重新登录引导；Aiotto 不要求用户提交账号密码或验证码。

### 中转管理与智能路由

- 管理 OpenAI-compatible、Anthropic-compatible 模型服务。
- 拉取并选择模型，查看余额、延迟、协议、网络模式和可用状态。
- 连通性测试、错误诊断、请求日志导出和配置导入导出。
- 官方模型与中转模型灵活选择，支持随时恢复官方使用路径。
- API 登录模式、中转模型显示标识和生图工具兼容设置。

### 会话、存储与备份

- 会话搜索、筛选、置顶、继续、归档、回收、恢复和导出。
- 长会话目录、用户与 AI 消息区分、宽窄窗口响应式阅读。
- 本地存储看板、会话占用分析和安全清理建议。
- 备份历史、恢复预览、恢复前保护和结果复检。

### 统计与状态

- 请求、token、缓存、模型来源和成本趋势。
- 官方与中转来源筛选，支持不同时间范围查看。
- OpenAI 主要服务状态、事故进展和恢复提醒。
- 应用内通知、macOS 系统通知、消息与公告。

### macOS 状态栏

- 不打开主窗口即可查看当前账号、额度、中转状态、本地存储和 OpenAI 服务状态。
- 支持显示内容、顺序、样式、颜色、隐私模式和点击行为配置。
- 提供刷新、设置、关于和退出等常用入口。
- 支持登录后后台启动；关闭主窗口后仍可保持必要的后台能力。

### 扩展、维护与设置

- Skills 与 MCP 状态盘点和健康信息。
- 系统诊断、安全清理、Codex 进程处理和常见异常恢复。
- 跟随系统、浅色、深色外观模式与多套主题色。
- 语言、登录启动、账号网络、刷新频率、通知动效和更新检查。
- 消息、公告和赞助支持入口。

## 典型使用流程

### 多账号与额度接力

1. 保存已经通过 Codex 正常登录的账号。
2. 在账号页查看各账号的 5 小时与每周额度。
3. 设置提醒阈值，或按需开启自动切换。
4. 额度不足时选择可用账号，确认后切换并重新进入工作状态。

### 添加并验证模型服务

1. 添加模型服务，填写名称、地址和自己的 API Key。
2. 拉取模型列表并选择需要使用的模型。
3. 先运行连通性测试，确认认证、协议、网络和首包响应正常。
4. 启用智能路由，在 Codex 中选择对应模型；不再使用时关闭即可恢复。

### 整理会话与释放空间

1. 在会话页找到需要保留、归档或恢复的任务。
2. 在存储页查看各类本地数据的真实占用。
3. 预览清理建议，核对影响范围和备份状态。
4. 执行后重新扫描，确认释放空间和剩余项目。

### 日常状态检查

1. 从状态栏快速查看账号额度、中转状态和 OpenAI 服务状态。
2. 收到额度或服务异常通知后，进入对应页面查看详情。
3. 需要完整操作时从状态栏打开工作台；只关闭窗口不会中断后台状态维护。

## 下载与安装

### 系统要求

- macOS 12 Monterey 或更高版本
- Apple Silicon 与 Intel Mac
- Universal 安装包

### 安装步骤

1. 打开 [最新 Release](https://github.com/ShawnZhuge/aiotto/releases/latest)。
2. 下载文件名以 `_universal.dmg` 结尾的安装包。
3. 打开 DMG，将 Aiotto 拖入“应用程序”文件夹。

当前安装包尚未完成 Apple Developer ID 签名与公证。首次打开时请在 Finder 中右键 Aiotto 并选择“打开”。如果 macOS 仍提示应用已损坏，可在确认安装包来自本项目官方 Release 后执行：

```bash
xattr -cr /Applications/Aiotto.app
```

## 常见问题

### Aiotto 会替我登录 Codex 吗？

不会。请先通过 Codex 的正常流程完成登录，再由 Aiotto 保存当前账号状态。Aiotto 不要求你提交账号密码、验证码或 Cookie。

### 使用中转模型后，还能回到官方模型吗？

可以。智能路由提供明确的启用与关闭流程；关闭后会恢复官方使用路径。切换前请阅读确认界面的影响说明。

### 为什么关闭窗口后 Aiotto 还在运行？

状态栏、额度提醒和智能路由等能力需要后台保持运行。点击窗口关闭按钮只隐藏主窗口；如需完全退出，请使用状态栏中的“退出 Aiotto”或按 `Command + Q`。

### 目前支持 Windows 或 Linux 吗？

暂不支持。目前只提供 macOS 12+ Universal 安装包。

### 遇到问题时应该提供什么信息？

请提供 macOS 版本、Mac 芯片类型、Aiotto 版本、复现步骤和脱敏后的错误信息。不要上传账号凭据、API Key、令牌或含有个人数据的截图与日志。

## 版本支持

Aiotto 目前处于 `0.x` 快速迭代阶段，只维护最新发布版本。遇到问题时，请先升级到 [最新版本](https://github.com/ShawnZhuge/aiotto/releases/latest) 再提交反馈。

每个正式版本都会提供 macOS DMG、应用内更新文件、版本说明和升级提醒。

## 从源码运行

本项目提供可独立构建的桌面应用，适合体验产品界面、学习桌面应用工程结构并参与改进。普通用户建议直接下载 GitHub Release。

### 本地运行

```bash
git clone https://github.com/ShawnZhuge/aiotto.git
cd aiotto
pnpm install
pnpm dev
```

### 构建检查

```bash
pnpm build
cargo check --locked --manifest-path src-tauri/Cargo.toml
pnpm tauri:build
```

### 项目结构

```text
src/          React 界面
src-tauri/    Tauri 桌面应用
assets/       品牌与产品图片
```
